01 · Arquitectura

Híbrida: Python decide, LLM phrasea

La lógica clínica vive en código auditable; el LLM solo aporta naturalidad. Inspirada en Rasa CALM 2026 y la guía Anthropic para dominios regulados.

Capa Python (determinística)

  • triage_route · red flags críticos siempre tienen precedencia (bypass total).
  • detect_red_flags · escanea síntomas estructurados con NICE/AAP.
  • assess_urgency · clasifica critical/urgent/standard.
  • calculate_acetaminofen_dose · floor(weight × 15mg/kg).
  • _sync_questions_asked · anti-loop, marca campos extraídos como asked.

Capa LLM (Gemini 2.5 Flash)

  • conversation_manager · clasifica intent del último mensaje.
  • receptor · structured output sobre últimos 6 mensajes.
  • adaptive picker · cada turno elige pregunta natural.
  • 5 ramas conversacionales · prompts ≤2 frases.
  • recommendation/urgency · mensaje único corto.
▶ Las 7 ramas conversacionales en detalle
data

"38.5 axilar" → receptor extrae → triage decide.

emotional

"estoy temblando, qué hago" → empathy_response sin avanzar checklist.

user_q

"¿es grave doctor?" → answer_question con datos disponibles, escala si faltan.

evasion

Cambio de tema → reframe_question retoma sin reproche.

mixed

"tiene 3 años, ¿qué le doy?" → receptor extrae → triage detecta pending → answer_question.

closing

"gracias doctor" tras recomendación → close_conversation.

scope_exit

Padre dice "no es fiebre" → mensaje de derivación, no insiste.

03 · Estado del arte

Por qué esta arquitectura

Cada decisión arquitectural está respaldada por investigación 2025-2026.

🚨

LLMs solos sub-trían urgencias

ChatGPT Health undertria 52% de casos de emergencia, mandando pacientes con DKA y fallo respiratorio inminente a "atención diferida". — Nature Medicine, "ChatGPT Health performance in a structured test of triage recommendations" (Jan 2026)

Decisión: la lógica de urgencia vive en Python determinístico — no en el LLM. triage_route bypassea cualquier turno con red flag crítico.

🛡️

Guardrails en system design

In a regulated industry, guardrails cannot just be prompt instructions—they must be embedded in the system design. — Anthropic, "Building Effective Agents" + AI Agents 2026 Practical Architecture (Apr 2026)

Decisión: dosis con floor(weight × 15mg/kg) en Python. Red flags con regex + negation-aware. Anti-loop mecánico.

🏥

Hartford PatientGPT pattern

Hartford HealthCare partners with K Health to launch PatientGPT, an AI agent for medical records that uses an independent review agent to validate clinical output before patients see it. — HIT Consultant, Hartford HealthCare K Health PatientGPT (Mar 2026)

Decisión: arquitectura con clasificador de intent + 6 ramas especializadas. La rama paralela agentic-architecture añade un reviewer node con invariantes globales.

💝

Empatía situada como criterio

Users prioritize coherence, contextual relevance, and empathetic understanding when evaluating clinical dialogue systems. — "Can Virtual Agents Care?" (arxiv 2604.20948)

Decisión: branch dedicada empathy_response que NO avanza el checklist. Detección de emoción dominante por turno.

🎯

Routing determinista > LLM

Deterministic routing is more reliable than asking an LLM to route. Use deterministic checks first; LLM-based routing only for genuinely ambiguous cases. — LangChain, LangGraph workflows-agents docs (2026)

Decisión: triage_route es Python puro. conversation_manager usa LLM con structured output (no free text).

⚡

"LLM no tan inteligente"

Gemini 2.5 Flash fue elegido a propósito: modelo intencionalmente económico cuya supervisión clínica recae en código Python. La consigna fue que "el assesment lo debe hacer un LLM no tan inteligente" — fuerza que la lógica crítica esté en código auditable, no en un LLM cerrado.
